# Data Accuracy & Integrity Rule (데이터 정확성 원칙)

## 📌 핵심 원칙 (Core Principles)

1. **100% 실측 데이터 기반 (Empirical Data Integrity)**
   - 플랫폼 내 모든 시세, 캔들 차트, 통계 데이터는 오직 실제 수집(Scraped/API)된 원본 실측치에만 근거해야 한다.
   - UI 차트 모양이나 일봉 렌더링을 보기 좋게 보이게 하기 위해 가짜(Synthetic/Mock) 날짜 패딩이나 임의 시세를 생성하여 덧붙이는 행위를 엄격히 금지한다.

2. **동일 수집 파이프라인 일관성 (Deterministic Pipeline)**
   - 동일한 프로바이더/항목에 대해서는 항상 단일화되고 일관된 동일 알고리즘 및 표준 단위(예: 인스턴스 전체 가격이 아닌 GPU 1개당 시간당 단가)를 적용한다.
   - 크롤링 방식(Live API vs Scraper)이 매번 바뀌거나 임의 전환되어 사용자가 데이터 신뢰성에 혼란을 느끼지 않도록 수집 파이프라인의 명확성과 수식 단위를 보장한다.

3. **투명한 데이터 품질 관리 (Quality Gate & Transparency)**
   - 수집 대상 원본 데이터에 0원, 음수, 혹은 노이즈 데이터가 포함되어 있더라도 시스템이 거짓 데이터를 임의로 지어내지 않는다.
   - 품질 게이트(Quality Gate)에서 0원/비정상 데이터를 명확하게 정제 및 격리(Quarantine)하여 데이터 신뢰도를 100% 사수한다.
